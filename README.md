# Задание 1
## Задача 10
### Текст задачи
Получить последовательность из произведений цифр соответствующих чисел, хранящихся в
исходной последвательности.
### Алгоритм решения
Программа запрашивает количество элементов и проверяет корректность ввода через Int32.TryParse.
Если n > 0, создается ленивая последовательность originalSeq с помощью генератора seq {...}.
Рекурсивная функция askForNumber:Запрашивает ввод числа у пользователя.
Базовый случай: если введено целое число, возвращает его значение.
Рекурсивный шаг: при ошибке ввода выводит предупреждение и вызывает саму себя для повторного запроса.
Функция productOfDigits:
Базовый случай: если число равно 0, возвращает 0.
Рекурсивный процесс (loop): отделяет последнюю цифру числа через % 10, умножает её на текущий аккумулятор и передает остаток числа через / 10 в следующий вызов.
Программа применяет Seq.map, привязывая функцию вычисления произведения цифр к каждому элементу последовательности.
С помощью Seq.iteri запускается выполнение последовательности: программа поочередно запрашивает ввод каждого числа, вычисляет произведение его цифр и сразу выводит результат на экран.

# Задание 2
## Задача 10
### Текст задачи
Последовательность содержит строки, обозначающие двоичные числа от 1 до 9. Найти в
десятичной системе счисления сумму элементов последовательности.
### Алгоритм решения
Программа запрашивает у пользователя количество элементов и пытается преобразовать ввод в целое число с помощью Int32.TryParse.
Если введено положительное число то создается последовательность binaryNumbers с помощью Seq.init, которая будет генерировать элементы по запросу.
Вспомогательная функция tryParseBinary:Пытается интерпретировать строку как число в двоичной системе счисления. Проверяет, входит ли результат в диапазон от 1 до 9. Возвращает значение, обернутое в Some, если проверки пройдены, или None в случае любой ошибки или выхода за границы.
Рекурсивная функция getValidBinaryInput:
Выводит запрос на ввод конкретного элемента последовательности.
Считывает строку из консоли и передает её в tryParseBinary.
Базовый случай: если функция парсинга вернула значение, оно возвращается как результат.
Рекурсивный шаг: если парсинг не удался, выводится сообщение об ошибке и функция вызывает саму себя для повторного запроса того же элемента.
Программа сначала проходит по последовательности с помощью Seq.iter, выводя все принятые числа (уже в десятичном виде) на экран. Затем с помощью Seq.fold вычисляется сумма всех элементов:
Процесс начинается с аккумулятора, равного 0.
На каждом шаге к текущему значению аккумулятора прибавляется следующее число из последовательности.
В конце выводится итоговое значение суммы.

# Задание 3
## Задача 10
### Текст задачи
Выяснить, есть ли в каталоге и его подкаталогах файл с введенным
названием.
### Алгоритм решения
Программа считывает название файла и начальный путь, проверяя существование указанного каталога с помощью Directory.Exists.
Если путь корректен, инициируется последовательность через вызов рекурсивной функции findFile.
Рекурсивная функция findFile:
Сначала функция перебирает все файлы в текущем каталоге через Directory.EnumerateFiles.
С помощью Seq.filter она отбирает только те пути, в которых имя файла совпадает с искомым, и возвращает их в общий поток через yield!.
Затем функция получает список всех подкаталогов через Directory.EnumerateDirectories.
Для каждого найденного подкаталога функция вызывает саму себя, а оператор yield! «склеивает» результаты из глубин файловой системы в единую последовательность.
После завершения обхода программа проверяет полученную последовательность на пустоту с помощью Seq.isEmpty.
Если файлы найдены, программа проходит по последовательности через Seq.iter и выводит каждый полный путь на экран. В противном случае выводится сообщение, что файл не найден.
